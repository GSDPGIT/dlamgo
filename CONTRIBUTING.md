# 贡献指南

欢迎提交 Issue、PR 或文档改进。

## 一、建议的贡献方向

- 修复 bug
- 改进文档
- 优化部署体验
- 优化 Go 后端结构
- 改进前端交互
- 改进发布与 CI 流程

## 二、提交前建议

请尽量先完成以下检查：

1. 后端可编译
2. 前端可构建
3. Docker 镜像可构建
4. 关键文档已同步更新

## 三、代码风格建议

- 优先保持当前项目结构一致
- 不要顺手改动无关文件
- 如果做了兼容性变更，请写清楚影响范围
- 如果做了破坏性变更，请在 PR 中明确说明

## 四、提交 Issue

提交 Issue 前建议先阅读：

- `README.md`
- `DOCKER.md`
- `doc/仓库结构说明.md`
- `doc/发布流程建议.md`

## 五、提交 PR

建议在 PR 中说明：

- 变更目的
- 主要改动
- 是否影响部署
- 是否影响数据库
- 是否更新了文档
