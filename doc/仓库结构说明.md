# 仓库结构说明

## 一、当前默认使用的目录

如果你的目标是部署或维护当前版本，请重点关注以下目录：

- `go-panel-backend`
  当前默认后端，技术栈为 Go + SQLite

- `vite-frontend`
  当前默认前端，技术栈为 Vite + React + TypeScript

- `go-gost`
  节点通信与控制逻辑目录

- `docker-compose.yml`
  当前默认 Docker Compose 部署入口

- `.env.example`
  Docker 部署环境变量模板

## 二、归档/兼容目录

以下内容仍保留在仓库中，但不再是默认部署路径：

- `springboot-backend`
  旧版 Java 后端，保留用于参考、迁移和历史兼容

- `android-app`
  Android 客户端工程

- `ios-app`
  iOS 客户端工程

- `docker-compose-v4.yml`
  历史 Compose 配置，保留用于兼容或参考

- `docker-compose-v6.yml`
  历史 Compose 配置，保留用于兼容或参考

- `panel_install.sh`
  历史面板安装脚本

- `install.sh`
  历史节点安装脚本

## 当前仍然保留的脚本入口

- `scripts/install-node.sh`
  当前默认节点安装脚本

## 三、其他文件说明

- `README.md`
  仓库主说明文档，优先阅读

- `DOCKER.md`
  Docker 部署快速说明

- `gost.sql`
  历史 MySQL 初始化文件，当前 Go + SQLite 默认方案不再依赖它

- `flux.ipa`
  历史产物文件，不参与当前默认部署

## 四、推荐阅读顺序

如果你是首次接触本项目，建议按以下顺序阅读：

1. `README.md`
2. `DOCKER.md`
3. `doc/仓库结构说明.md`
4. `doc/发布流程建议.md`
